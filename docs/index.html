So... how does this work?

Introduction

OpenGroup is a system in which you can have groups to communicate with people over the internet. Everybody can create plugins and different groups can have different plugins.

  • p2p based
  • Open plugin system
  • Open source

OpenGroup exists out of a couple of components:

  • OpenGroup specification
  • OpenGroup Core
  • Hosted OpenGroups
  • Plugins and themes
  • Plugin and theme repositories

It would be great if a lot of people would have control over their data again. With p2p networking we can make this possible. OpenGroup does not need servers so no advertisements and no stakeholders.

Group Specification

At the core of OpenGroup is a specification with describes how you can created an HTML file which is an OpenGroup. This HTML may be hosted, it may be generated or hand crafted. Inside the HTML file is something like a script tag which holds a json structure in which the plugins and other things are defined.

It could look something like this:

window.opengroup = {
    "type": "opengroup",
    "version": "2.1",
    "core": "https://cdn.opengroup.io/core/2.1/core.js",
    "plugins": [
        "https://cdn.opengroup.io/plugins/messaging/2.1/messaging.json"
    ],
    "theme": "https://johnsopengroupthemes.com/themes/dark.json",
    "permissions_type": "owner",
    "permissions_data": {
        "owner": "ssh-rsa AAAAB3NzaC1yc2EA..."
    }
}

The HTML file should have something like an import tag. See html5rocks for more information. This import tag could look a bit like this.

<link rel="import" href="https://cdn.opengroup.io/bootstrap.html">

Inside this bootstrap.html are the javascript files that parse the window.opengroup parameter and start the group.

OpenGroup Core

The Core of OpenGroup is an HTML file which can be hosted everywhere, but probably most people would use the cdn version. In this HTML file are a couple of small javascript programs. One needs to parse the group json, another needs to bootstrap the app. OpenGroup Core is a framework. There should be good documentation available for creating plugins and themes. The framework should have hooks, events or triggers. A plugin definition could be done as follows:

https://cdn.opengroup.io/plugins/messaging/2.1/messaging.json

{
    "menu-item": {
        "title": "Chat",
        "files": [
            "messaging.core.js",
            "messaging.bus.js",    
        ]
    }
}

The framework should abstract away all the difficult p2p things. Sending a message to all group members should be as easy as:

opengroup.emit({
    "command": "plugin.messaging.message",
    "data": "Hello friends!"
});

The framework should support optimisation of connections. For example with a group of 5 members it is fine to connect everyone to everyone, but with groups of 100 people that would mean a lot of connections. It would be better to split up in sort of sub groups.

WebRTC

WebRTC is a protocol to connect browsers to each other via p2p. WebRTC makes it possible to share files, video chat or voice call people via the browser without extra plugins.

Signaling

Signaling is process of connecting two browsers via WebRCT.
